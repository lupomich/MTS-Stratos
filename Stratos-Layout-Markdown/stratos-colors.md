# Stratos Design System - Colors

## Color Fundamentals

### Color Theory
Un colore è definito da 3 caratteristiche:
- **Hue** (Tonalità): la posizione sul cerchio cromatico
- **Saturation** (Saturazione): l'intensità del colore
- **Lightness** (Luminosità): quanto chiaro o scuro è il colore

### Color Nomenclature
Nel sistema Stratos, ogni variabile colore è composta da:
- **Token**: il nome del colore (es. Blue, Teal, Grey)
- **Index**: il tono/variante del colore (es. -10, -20, -30, -40, -50)

**Esempio:**
- `Blue - 30` → Color token: Blue, Index: 30
- `Teal - 50` → Color token: Teal, Index: 50
- `Grey - 10` → Color token: Grey, Index: 10

## Usage Guidelines

### Do's ✅
- Usa **Teal - 40** (`#008D7F`) come colore primario per azioni principali
- Usa i colori semantici per feedback (success, error, warning, info)
- Mantieni contrasti accessibili (WCAG AA minimum)
- Usa Grey scale per testi e backgrounds neutri

### Don'ts ❌
- Non usare colori brand per stati semantici (es. Teal per errori)
- Non mixare troppi colori nella stessa interfaccia
- Non usare colori con contrasto insufficiente per il testo
- Non creare nuove varianti di colore fuori dalla palette
